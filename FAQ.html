Frequently Asked Questions

Which version of Spray is riak-scala-client compatible with?

Version {{ site.version }} of riak-scala-client was developed in the vacuum between Spray 1.1-M7 and 1.1-M8. The spray-client module was heavily refactored early in the 1.1-M8 timeframe and we used the resulting new features expecting a quick release of 1.1-M8. However the Spray team then (February 2013) decided to start a massive refactoring and redesign process and as a result 1.1-M8 has not been released yet (mid-March 2013).

While we all wait patiently for Spray 1.1-M8 to be released, riak-scala-client depends on a nightly build of Spray 1.1, namely 1.1-20130123. This version is very close to 1.1-M7 but includes the above-mentioned first refactoring of spray-client.

As soon as Spray 1.1-M8 (or later) is released, a compatible version of this library will be released ASAP.

All of this will not affect you very much unless you want to use this library in a project that depends on Spray 1.1-M7, esspecially spray-client. In that case unfortunately the only available choices are either to upgrade to the same nightly build of Spray or to not use this library until Spray releases 1.1-M8.

Will there ever be support for the Riak protobuf API?

You never know, but probably not. It is certainly not a priority for 1.0 unless lots of people indicate 1.0 would not be complete without it.

If you are one of those people, please make your wishes know by creating an issue or a pull request!.

Why such a boring name?

It seems all the cool and appropriate names, like riaktive, riakka, riaktor, scalariak, etc. have already been taken by other projects. But there seems to be a common riak-xxx-client naming pattern used by Riak client libraries for other languages so that's what ended up deciding the, admittedly boring, name.

If you come up with a cooler name, please let us know and eternal fame will be yours!
